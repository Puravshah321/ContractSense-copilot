# ContractSense Copilot - Deployment Guide

## Quick Start (Local Preview)

```bash
# 1. Activate your environment
.\contractsense_env312\Scripts\activate

# 2. Install streamlit (if not already installed)
pip install streamlit

# 3. Run the app
streamlit run app/main_app.py
```

The app will open in your browser at `http://localhost:8501`.

## Deploy to Streamlit Cloud (FREE Hosting)

### Step 1: Push Code to GitHub
Make sure your `jay` branch is up to date:
```bash
git add .
git commit -m "feat: Stage 8 - ContractSense Legal Copilot UI"
git push origin jay
```

### Step 2: Go to Streamlit Cloud
1. Open [https://share.streamlit.io](https://share.streamlit.io)
2. Sign in with your **GitHub account** (the one that owns the repository)

### Step 3: Deploy
1. Click **"New app"**
2. Fill in the details:
   - **Repository:** `Puravshah321/ContractSense-copilot`
   - **Branch:** `jay`
   - **Main file path:** `app/main_app.py`
3. Under **Advanced settings**, set:
   - **Python version:** `3.12`
   - **Requirements file:** `requirements_app.txt`  
     *(NOT `requirements.txt` -- that one has GPU dependencies that will fail on Streamlit Cloud)*
4. Click **"Deploy!"**

### Step 4: Wait for Build
Streamlit Cloud will:
1. Clone your repo
2. Install the dependencies from `requirements_app.txt`
3. Start the app

This usually takes **2-3 minutes**. Once ready, you will get a public URL like:
```
https://contractsense-copilot.streamlit.app
```

## How the Demo Mode Works

Since Streamlit Cloud does not have GPU access, the app runs in **Demo Mode**:

- It uses **pre-computed responses** from `app/demo_data.py`
- These responses were generated by our actual DPO-aligned model on Lightning AI
- The demo covers **6 clause types** with **3 queries each** (18 total interactions)
- Custom/free-text queries receive a generic analysis response

This approach lets you showcase the full pipeline without needing a $500/month GPU server.

## App Structure

```
app/
  main_app.py       # Main Streamlit application (3 pages)
  demo_data.py      # Pre-computed DPO model responses
.streamlit/
  config.toml       # Dark theme + server configuration
requirements_app.txt # Lightweight deps for Streamlit Cloud
```

### Pages
1. **Clause Analyzer** - The main copilot interface. Select a clause type, pick a question, and get structured RISK / EXPLANATION / ACTION / CITATION output.
2. **Performance Dashboard** - Visual comparison of Stage 6 (SFT) vs Stage 7 (DPO) metrics with error elimination stats.
3. **About** - Full pipeline architecture walkthrough (Stage 1 through Stage 8).

## Troubleshooting

### "ModuleNotFoundError: No module named 'app.demo_data'"
Make sure you are running from the project root:
```bash
streamlit run app/main_app.py
```

### Streamlit Cloud build fails
- Ensure you selected `requirements_app.txt` (NOT `requirements.txt`)
- The full `requirements.txt` contains `torch`, `transformers`, etc. which are too large for Streamlit Cloud's 1GB memory limit

### App looks different locally vs on Cloud
- The dark theme is defined in `.streamlit/config.toml`
- Make sure this file is committed and pushed to your branch

## Future: Full GPU Deployment

When you are ready to deploy the **real model** (not demo mode), you have two options:

### Option A: HuggingFace Spaces (Free GPU)
1. Create a new Space on [huggingface.co/spaces](https://huggingface.co/spaces)
2. Select **Streamlit** as the SDK
3. Upload your code + model adapters
4. HuggingFace provides free T4 GPU access for Spaces

### Option B: Cloud VM (Full Control)
1. Rent a GPU VM (AWS g4dn.xlarge, GCP n1-standard-4 + T4, or Lambda Labs)
2. Install dependencies from `requirements.txt`
3. Run `streamlit run app/main_app.py --server.port 80`
4. Point your domain to the VM's IP address
